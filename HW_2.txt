-------------------------------------------ОТВЕТЫ--------------------------------------------
1.git restore filename    (отменяет окончательно изменения в выбранном файле)

2. git reset HEAD имяфайла.

3.git reset --soft HEAD^

4.git checkout -b [branch name]

5.На сколько я понял, процесс слияния(megre) может проходить в двух режимах(ff /nff).
	no fast-forward - Хранит всю информацию о слияниях веток.(подход может оказаться запутанным и сложным, необходимо читать историю коммитов)
	fast-forward    - Хранит не всю инфу о слияниях веток.(просто для чтения, но в этом случае история веток проекта неочевидна)

6.С одной стороны ситуация проста, а с другой стороны зависит от стадии.Конфликт слияния может возникнуть либо ВНАЧАЛЕ либо в ПРОЦЕССЕ оного.
	Как я понял ситуация достаточно частая, так как возникает в проектах где работает большего одного разраба(другими словами два разработчика пытаются
	работать с контентом ОДНОВРЕМЕННО).
	git status   - пытаемся идентифицировать конфликтующие файлы
	git log --merge   - создаем журнал со списком конфликтов коммитов между ветками(для которых делается слияние)
	git diff 	  - ищем различия между состояниями файлов/репозитория.
	gti checkout 	  - можно использовать для отмены изименений в файлах или ветках.(до начала слияния)
	git megre --abort    - команда прерывающая процесс слияния(ветка вернется к состоянию ДО слияния)
	git reset 	- еще одна команда для разрашения конфликтов.(возникающих во время слияния)

7. Да можно,  через веб портал.
	      git branch -d [branch name]
